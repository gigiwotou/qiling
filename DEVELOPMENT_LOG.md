# PC Assistant 开发记录

## 项目概述
PC Assistant 是一个基于 Rust 开发的智能助手系统，包含后端核心功能和前端 UI 界面。

## 开发环境
- 操作系统：Linux
- 开发语言：Rust
- 前端框架：Tauri (Rust + WebView)
- 版本控制：Git + GitHub

## 技术架构

### 后端架构
- **模块化设计**：每个功能模块独立封装
- **异步处理**：使用 Tokio 进行异步操作
- **通信系统**：基于频道的模块间通信
- **配置管理**：基于 TOML 的配置系统
- **API 集成**：集成多个 AI 服务提供商

### 前端架构
- **Tauri 框架**：基于 Rust 和 WebView
- **响应式设计**：适配不同屏幕尺寸
- **本地集成**：与系统功能深度集成

## 遇到的问题与解决方案

### 1. GitHub 大文件上传限制
- **问题**：构建产物超过 GitHub 100MB 文件大小限制
- **解决方案**：清理 git 历史中的大文件，添加 .gitignore 排除构建产物

### 2. 仓库迁移
- **问题**：需要从一个仓库迁移到另一个仓库
- **解决方案**：更新远程仓库 URL 并强制推送

### 3. 权限配置
- **问题**：新工具需要添加到权限配置中
- **解决方案**：更新默认配置和权限配置，确保新工具被正确授权

## 贡献指南

### 代码规范
- 遵循 Rust 代码风格指南
- 使用 meaningful 的变量和函数命名
- 添加适当的注释
- 保持代码简洁和可维护

### 提交规范
- 使用清晰的提交信息
- 一次提交只包含一个功能或修复
- 提交前运行测试

### 开发流程
1. 克隆仓库：`git clone https://github.com/gigiwotou/qiling.git`
2. 安装依赖：`cargo build`
3. 开发新功能
4. 运行测试：`cargo test`
5. 运行自动推送脚本：`./auto_push.sh`
